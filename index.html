    ( ワーク名・品番・測定項目 )


    2. 品番・測定項目に「change」イベント処理

    ( 両方のイベント処理関数をリファクタリング )

  • 【 前回複写機能 】

    1. " localStorage " を使用


    2. inputタグに'required' で未入力箇所を表示

  • 【 イベントデリゲーション 】

    1. 親要素でイベントを発火させる


    2. 親要素から子要素や祖先を見つけて処理

    ( もっと良い方法を見つけたい・・・ )

～Rakujan Lv.2 (イテレーション2) ～

ユーザーストーリー・フローチャート

こだわり①

  • 【 ログイン画面実装 】

    1. 各ユーザーオブジェクト → 配列

    ( 名前・所属コード・電話番号 )


    2. findメソッドでinputデータと照合


    3. setTimeoutで非同期処理

    ( ユーザー名の表示 )

こだわり②

  • 【 UI / UX を追及 】

    1. 依頼時のUIを必要最小限


    2. 各ワークオブジェクト → 配列

    ( ワーク名・品番・各工程・目的・測定項目 )


    3. setTimeoutで非同期処理

    ( 測定依頼詳細の表示 )


    4. 依頼履歴通知

苦労したところ

  • 【 イベントデリゲーション 】

    1. 親要素でイベントを発火させる


    2. 親要素から子要素や祖先を見つけて処理

    ( もっと良い方法を見つけたい・・・ )


    【 自作スライダー 】

    リファクタリング → 個別にスライダーが動かない

    ( ワークオブジェクト分の配列を作成し何とか解消 )
